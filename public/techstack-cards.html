🛡️ Security Measures Applied
All hardening done during this build session — April 2026
🪖
helmet (security headers)CSP, X-Frame-Options, HSTS, X-Content-Type — blocks clickjacking & XSS
🌐
CORS AllowlistReplaced wildcard cors() with ALLOWED_ORIGINS env var — only your domains accepted
🚦
Rate Limiting200 req/15min on all API routes · 20 req/min on AI endpoints — blocks abuse
💰
Budget Guard (Daily Cap)Gemini: 1,400/day · Perplexity: 200/day — limits AI overuse and abuse
🔑
Gemini API Key RotatedExposed key removed from .env · New key generated · Old key revoked safely
🖼️
Vision MIME Type ValidationAllowlist: JPEG, PNG, WebP, GIF, PDF only — rejects malicious file uploads
🧹
XSS Fixed in ExtensionPopup rebuilt with createElement/textContent — no more innerHTML injection risk
🗑️
Anthropic SDK Removed@anthropic-ai/sdk & ai-agent.ts deleted — reduced attack surface & cost
📦
Basket Item CapMax 200 items × 200 chars each — prevents oversized payload DoS attacks
🔒
Image Size ValidationBase64 checked against 10MB max — rejects abuse before AI processing
🧽
Input SanitizationAll query params + body stripped of HTML & dangerous chars before any route
🚫
Suspicious Pattern GuardAuto-blocks path traversal, SQL injection, XSS, WordPress/PHP scanners
🗝️
Bearer Token Auth on AIAPP_API_KEY — strangers can't call /api/ai/* even if they find your URL
📏
Query Param Length GuardMax 300 chars per param — rejects fuzzing & buffer-overflow attempts
📋
Security Event LoggingEvery attack logged with timestamp, IP, path & pattern — know when you're being hit
🛡️ Security Score: 9 / 10
Missing 1 point: WAF (Web Application Firewall) — needed only at enterprise scale
